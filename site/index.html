what sunbeams does

One binary, three jobs.

[generate]

Synthesises the EDID

$ sunbeams generate

Packs every target resolution — 4K, ultrawide, MacBook, iPad, phones, handhelds — into a single EDID binary with HDR10 metadata and wide range limits. Outputs helper scripts for modes that exceed the EDID pixel-clock ceiling.

[install]

Installs on Bazzite

$ sudo sunbeams install

Scans /sys/class/drm for a free connector, writes the EDID to /etc/firmware/ (bypassing the immutable /usr), and injects firmware_class.path, drm.edid_firmware, and video= kernel arguments via rpm-ostree kargs.

[switch]

Switches the display

$ sunbeams switch on / off

Wires into Sunshine's Do/Undo prep commands. Reads SUNSHINE_CLIENT_*, snaps the request to the nearest configured mode, drives kscreen-doctor in one atomic call, and logs every decision to stderr.

quick start

End-to-end on a Bazzite box.

~/bazzite $ 
#  1. generate the EDID + helper scripts
$ sunbeams generate

#  2. install: EDID → /etc/firmware, kernel args via rpm-ostree, then reboot
$ sudo sunbeams install

#  3. inspect what the EDID exposes
$ sunbeams devices
$ sunbeams modes

#  4. wire into Sunshine → General → prep commands:
#     Do:   sunbeams switch on
#     Undo: sunbeams switch off
